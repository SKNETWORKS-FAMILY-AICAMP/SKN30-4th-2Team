# 프론트엔드 작업 내역 및 요구사항 반영 현황 (07/24)

* 작성일: 2026-07-24
* 결정 대상: 프론트엔드 UI 개선 및 Mock 연동, 정책 반영 공유
* 관련 문서:
  * 화면별_기능_정의서.md
  * GitHub Issue #11 ([FE] S01, S02·S03, S04, S05·S09)

## 1. 개요
백엔드 API 연동 전 프론트엔드 화면의 전반적인 완성도를 높이고, 팀장님이 공지하신 최신 협의 내용과 이슈 요구사항을 화면에 즉시 반영하기 위해 진행된 작업 내역을 공유합니다.

## 2. 핵심 변경 및 결정 사항

### 2.1 Mock API 기반 전체 흐름 검증 완료
- 백엔드 연동 대기 중 프론트엔드 플로우를 검증하기 위해 `mockApi.ts`를 도입했습니다.
- [업로드] → [유형 선택] → [검토 진행/폴링] → [결과 확인] 까지의 핵심 워크플로우를 모두 연결했습니다.
- **새로고침 대응**: `localStorage`에 `session_id`와 `review_id`를 보관하여 중간에 새로고침을 하더라도 현재 상태 화면으로 즉시 복구되도록 세션 연동 로직을 고도화했습니다.

### 2.2 최신 협의 정책(Issue #11) 프론트엔드 적용
- **계약 유형 고정**: 화면 내 선택 가능한 계약 유형을 `SW 프리랜서 용역`, `SI 하도급`, `SM 하도급` 단 3가지로만 고정했습니다.
- **파일 제약 조건**: 10MB 이상의 파일은 서버로 요청을 보내기 전 프론트엔드 단에서 원천 차단하도록 사전 검증을 추가했습니다.
- **결과 화면(`MISSING`) 분리**: `MISSING`(포함 여부 확인) 조항들은 일반 비교 결과 목록과 섞이지 않도록 아예 독립된 체크리스트 영역으로 분리하여 렌더링하도록 레이아웃을 수정했습니다.
- **빈 배열 처리**: 결과가 빈 배열일 때 "안전함" 같은 주관적인 표현을 지양하고 중립적인 문구로 안내되도록 처리했습니다.

### 2.3 글로벌 테마 및 UI 가독성 개편
- 디자인 일관성을 위해 앱 전체의 메인 테마 색상을 기존 Blue(`#2563EB`)에서 Indigo(`#6366F1`) 톤으로 전면 교체했습니다.
- 파일 업로드 컴포넌트의 가독성 개선(아이콘 확대, 텍스트 재배치)을 통해 사용자 이탈 및 오류 가능성을 줄였습니다.
- 상단 로고를 디자인 시안에 맞추어 `ShieldCheck` 아이콘 기반으로 업데이트했습니다.

## 3. 향후 계획 및 협의 필요 사항 (Next Step)
- **임시 UI 요소 및 기획 협의 필요 사항**:
  - 좌측 사이드바 하단에 임의로 배치해 둔 **"도움이 필요하신가요? (고객지원)"** 박스의 실제 유지 여부 및 연결될 링크(FAQ, 문의 채널 등)에 대한 기획 협의가 필요합니다.
  - 현재 시각적으로만 구현된 빈 상태(Empty State)나 세션 만료 등의 안내 문구에 대해 추가적인 워딩 리뷰가 필요합니다.
- **백엔드 연동 및 오류 처리 고도화**:
  - 실제 백엔드 API 배포 시, 현재 프론트엔드의 `mockApi.ts`를 즉시 걷어내고 실제 네트워크 요청으로 교체 및 SSE 폴링 테스트를 진행합니다.
  - 연동 실패, 타임아웃, 세션 만료 등의 오류 상황별 **재업로드 및 다시 시도** 버튼 노출 등 예외 처리(Fallback UI)를 고도화할 계획입니다.
